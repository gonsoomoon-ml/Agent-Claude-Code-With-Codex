# ④ Web UI — v1.1d 진행 모달 (Progress Modal) 설계 스펙 (2026-06-28, LANE B)

> 증분: v1.1c(inline StatusCard)의 UX 보강. 문제: inline 카드는 버튼 아래라 사용자가 놓침.
> 사용자 결정: 체험/구독 클릭 시 **중앙 모달(pop-up)** + **생성 중에도 항상 닫기 가능**("닫아도 메일로 도착").
> 범위: **프론트엔드 전용** — 백엔드/status API/폴링 로직(v1.1c) 무변경.

## 0. 결정
- inline `StatusCard` → **`ProgressModal`**(딤 backdrop + 중앙 dialog). 같은 진행 내용(스피너+경과타이머+status 카피)을 모달로.
- **항상 닫기 가능**: 닫기/Esc/배경클릭 → 닫힘 + **폴링 정지**(체험은 백그라운드 계속 → 메일 그대로 도착). 생성 중에도 닫기 활성.
- 접근성: `role="dialog"` `aria-modal="true"` · 오픈 시 닫기 버튼 포커스 · Esc 닫기.

## 1. 컴포넌트
- **신규 `web/src/components/ProgressModal.tsx`**: props `{title, text, busy, elapsedSec, onClose}`. fixed 딤 backdrop(클릭=onClose) + 중앙 카드(`role="dialog"`, stopPropagation) — 제목·스피너(busy)·text·경과타이머(busy)·"💡 창을 닫아도 계속 생성돼 메일로 도착합니다."(busy)·**[닫기]**(focus on open). Esc 리스너.
- **`web/src/pages/Form.tsx`**: inline `{card && <StatusCard.../>}` → `{card && <ProgressModal title="브리핑 체험"/"구독" text={card.text} busy={card.busy} elapsedSec={card.elapsed} onClose={() => { stopPolling(); setCard(null) }} />}`. 체험/구독 클릭 → card set(모달 오픈). 폴링이 card 갱신 → 모달 내용 실시간.
- **`StatusCard.tsx` 제거**(ProgressModal 이 대체; Form 만 사용했음). `trialStatus.ts`(순수) 유지.

## 2. 테스트
- **web**(vitest): `ProgressModal` — 렌더(dialog role·text·닫기) · 닫기 클릭→onClose · Esc→onClose · busy 시 타이머/안내 표시.
- **Form 테스트 갱신**(`Form.progress`·`Form.trial`·`Form.subscribe`): inline `getByRole('status')` 단언 → **`getByRole('dialog')`** + 동일 text 단언(약화 금지, 모달로 이전). 신규: 생성 중 닫기 → dialog 사라짐 + 폴링 정지(추가 status fetch 없음).

## 3. 배포 / e2e
- deploy_web 재배포. e2e: moongons 체험 → **모달 팝업** → 생성 중 닫기 가능 확인 → (안 닫으면) 단말 상태 표시.

## 4. 미룸
- 모달 애니메이션(fade) · 포커스 트랩(현재 닫기 포커스+Esc 로 충분) · 토스트 알림 대안.
