# ④ Web UI — v1.1e 테스트 주소 쿨다운 우회 (TRIAL_TEST_EMAILS) 설계 스펙 (2026-06-28, LANE B)

> 증분: 자주 테스트하려면 v1.1a 체험하기의 **1시간 쿨다운**에 막히거나(429 "최근에 이미 요청했어요") 컨트롤러가 매번 DDB 행을 지워야 함.
> 사용자 결정: `.env` allowlist(`TRIAL_TEST_EMAILS`)로 **내 테스트 주소만** 쿨다운 우회. **공개 사용자 가드(쿨다운+cap)는 무변경.**
> 범위: **백엔드 전용** — 프론트/모달/폴링/status API 무변경.

## 0. 결정
- `TRIAL_TEST_EMAILS`(쉼표 구분 이메일) → allowlist 주소만 `within_cooldown` **게이트 skip**.
- **전역 cap(50/일, `GLOBAL#<date>` 카운터, 하드코딩)은 테스트 주소도 계속 카운트** → 절대 비용 천장 유지. cap 은 **env 패스스루 금지**(현 원칙 보존).
- **행 기록(`record(status, ttl)`)은 유지** → 모달 status 폴링(`get_status`)이 행을 읽어 `생성 중→발송`을 따라가야 함. 쿨다운 *검사*만 skip, *기록*은 동일.
- **미설정/빈값 → 빈 frozenset → 오늘 동작과 byte-identical**(안전 기본값; 비협상 불변식).
- 우회 판정은 **소문자·trim 정규화 후 멤버십** — `handle_trial` 이 이미 `email = (...).strip().lower()`(trial.py:64) 하므로 allowlist 도 동일 정규화.

## 1. 컴포넌트

### 1a. `webapi/trial.py`
- **신규** `_parse_emails(raw: str) -> frozenset[str]`: `raw.split(",")` → 각 `strip().lower()` → 빈 문자열 제거 → `frozenset`. (`""` → `frozenset()`.)
- `handle_trial(payload, *, store, ses, runtime_invoke, cap, cooldown_s, today, catalog_keys, test_emails: frozenset[str] = frozenset())` — 새 **키워드 인자(기본 빈 set)**. 쿨다운 분기(현 trial.py:69)를
  ```python
  if email not in test_emails and store.within_cooldown(email):
      return 429, {"error": "최근에 이미 요청했어요. 잠시 후 다시 시도하세요."}
  ```
  로 변경. **그 외(validate·over_global_cap·SES verify·record·runtime_invoke·반환)는 무변경.** 기본 인자라 기존 호출부(테스트 포함)는 무손상.

### 1b. `webapi/app.py`
- `_trial_deps()` 반환 dict 에 `"test_emails": _parse_emails(os.getenv("TRIAL_TEST_EMAILS", ""))` 추가(`_parse_emails` import).
- `post_trial` 의 `handle_trial(...)` 호출에 `test_emails=d["test_emails"]` 한 인자 추가. (나머지 호출 인자 무변경.)

### 1c. `webapi/deploy_api.py`
- `_deploy_lambda` 의 Lambda `env["Variables"]` dict 에 `"TRIAL_TEST_EMAILS": os.getenv("TRIAL_TEST_EMAILS", "")` **pass-through** 추가(배포 호스트 `.env` → Lambda env). 빈 기본값.
- `"TRIAL_GLOBAL_CAP": "50"` 및 그 "env 패스스루 금지" 주석은 **그대로 유지**(cap 은 절대 천장).

## 2. 테스트 (pytest, fake store/ses — `tests/test_webapi_trial.py` 패턴)
- **allowlist + 쿨다운중** → `within_cooldown` 가 True 여도 **202**(쿨다운 skip 확인).
- **비-allowlist + 쿨다운중** → **429**(공개 가드 유지 — 회귀 방지).
- **allowlist + over_global_cap True** → **429**(cap 은 테스트 주소도 차단 = 천장 유지).
- **빈 allowlist(인자 미전달/`frozenset()`)** → 쿨다운 enforced(오늘 동작과 동일).
- **`_parse_emails` 정규화** — `"A@x.com, b@y.com ,"` → `{"a@x.com", "b@y.com"}`; `""` → `frozenset()`.

## 4. 보안 / 가드 (불변식)
- **공개 경로 무손상**: 비-allowlist 는 쿨다운 + cap 그대로.
- **비용 천장 유지**: 전역 cap(50/일) 하드코딩 + allowlist 주소도 카운트.
- allowlist 는 **gitignored `.env`/Lambda env 에만** — 코드·커밋·공개 노출 없음.
- **잔여 위험(수용·문서화)**: 누군가 *소유자 주소*로 trial 제출 시 쿨다운 우회로 **소유자 받은편지함**에 브리핑 더 자주 도착 가능 — cap(50/일) + 카드 캐시로 바운드, blast radius = 소유자 본인. 데모 수용.
- **신뢰경계**(`skill_md` 미전송)·**IDOR**(recipient = 제출 이메일) 무변경.
