## 0. 결정 (한 줄)
Cognito **PKCE 로그인 → id_token → `PUT /profile`(JWT-gated) → `DynamoUserStore.update_profile_from_jwt`(PK=Cognito `sub`, recipient=Cognito `email`) → ⑤ 가 `load_user(dynamo)`로 자동 픽업.** **PK·recipient 는 검증된 JWT claims 에서만**(request body 절대 금지). v1.2 는 H3/H4 를 새 코드 변경 없이 소비 — **쓰기 경로 + 인증만** 추가.

## 1. End-to-end flow (검증된 경로)
```
[/setup 폼] 출처·시각 → [구독하기]
  └ 미인증: startLogin() → {hostedUI}/oauth2/authorize (code+PKCE S256, scope=openid email, state)
       └ 콜백 ?code&state → state 검증 → /oauth2/token (grant_type=authorization_code, client_id, code_verifier, NO secret)
            → {id_token} (in-memory 보관)
  └ 인증: PUT /profile  Authorization: Bearer <id_token>   body=6 선호필드(type,sources,depth,lens,send_hour,timezone)
       └ HTTP API 네이티브 JWT authorizer (issuer=pool, audience=client_id, **id_token만**) 검증
            └ Lambda: scope["aws.event"]["requestContext"]["authorizer"]["jwt"]["claims"] → sub·email (body 무시)
                 → validate(6필드) → UpdateItem briefing-users(Key={user_id: sub}, recipient=email, 6필드)
                 → _ensure_ses(email): SES 미검증이면 verify_email_identity → delivery: active|verification_pending
[GET /profile] (JWT) → 기존 구독 prefill {subscribed, recipient=email, profile}
  ⑤ 07:00 KST: list_users(Scan) → load_user(get_user + 파일 skill_md "") → due면 author→certifier→SES send(recipient)
```

## 2. 컴포넌트 (정확한 형태)

### 2a. web — `web/src/auth/*.ts` (no amplify, 순수 TS)
- `config.ts`(hostedUI·clientId·issuer·redirectUri origin분기·scope="openid email") · `pkce.ts`(verifier/challenge S256 via Web Crypto·state, base64url no-pad) · `login.ts`(startLogin: verifier+state→**sessionStorage**→authorize redirect) · `callback.ts`(handleCallback: **state 검증 필수**·token 교환·`history.replaceState`로 ?code 제거) · `session.ts`(id_token **in-memory**·`authedFetch`→Bearer) · `logout.ts`.
- `api.ts` +`getProfile()`·`putProfile(p)` (authedFetch).
- `App.tsx`: 마운트 useEffect 에서 `?code` 있으면 handleCallback→storeTokens. Form: 미인증=로그인 버튼, 인증=폼.
- `Form.tsx`: 기존 SourcePicker/시각 재사용. **이메일 입력 제거**(JWT email 사용) → "구독 주소: {email}" 표시. **구독하기** 버튼 활성(인증+≥1출처) → putProfile → 상태(active/verification_pending/error). 마운트 시 getProfile prefill(재구독=수정).

### 2b. api — `webapi/profile.py`(신규) + `app.py` 수정
- `profile.py`: `validate_profile(fields, *, catalog_keys, lens_keys, depths, send_hours, types=("ai-news",)) -> str|None` — sources 1~5∧⊆catalog · type∈types · depth∈DEPTHS · lens∈lens_keys · send_hour∈SEND_HOURS · timezone=str(+ zoneinfo 검증). (catalog enums 는 catalog.py 재사용.)
- `app.py`: `_claims(request)` 헬퍼 — `request.scope.get("aws.event")` → claims 추출, `token_use!="id"`→401, sub/email 없으면 401(방어심도). `GET /profile`(prefill) · `PUT /profile`(validate→update_profile_from_jwt→_ensure_ses). `_profile_deps()` lazy boto3(store=DynamoUserStore·ses·sender·keys/lenses). `_ensure_ses` = trial 의 get_identity_verification_attributes→미검증 시 verify_email_identity→"active"|"verification_pending".
- **CORS**: allow_methods 에 **PUT** 추가, allow_headers 에 **Authorization** 명시.

### 2c. dynamo — `DynamoUserStore.update_profile_from_jwt(*, sub, email, fields)` (IDOR-safe)
`UpdateItem(Key={user_id: sub}, SET recipient=email + 6필드)`. **모든 필드 `#name` 우회**(★ `type` 은 DDB 예약어). sources→list·send_hour→int. body 의 user_id/email/recipient 절대 미사용. `put_user`(admin 시드)는 ④ 에서 호출 금지.

### 2d. deploy_api — JWT authorizer + /profile 라우트 + IAM
- IAM: `BriefingTrial` 정책에 `UsersTable` statement 추가(`dynamodb:GetItem,UpdateItem` on `table/briefing-users`).
- `_ensure_http_api`: idempotent `create_authorizer`(Name=cognito-jwt, Type=JWT, IdentitySource=`$request.header.Authorization`, Jwt={Issuer=`https://cognito-idp.us-east-1.amazonaws.com/us-east-1_ANfcEK61A`, Audience=[client_id]}) + `GET /profile`·`PUT /profile` 라우트(AuthorizationType=JWT, AuthorizerId). **$default 는 NONE 유지**(catalog/sample/trial/health 공개). **OPTIONS /profile 라우트 선언 금지**(preflight 는 $default→CORSMiddleware 가 토큰 없이 처리).

## 3. 보안 불변식 (구현이 깨면 안 됨)
| 불변식 | 근거 |
|---|---|
| **PK(user_id) = jwt.claims.sub** — body 금지 | IDOR/프로필 탈취 차단 |
| **recipient = jwt.claims.email** — body 금지 | SES 스팸 릴레이 차단(우리 sender 로 임의 발송) |
| **id_token만 Bearer**(access token 불가) | Cognito access token 엔 aud 없어 authorizer 401 |
| **token_use=="id" 핸들러 assert** | authorizer drift 방어 |
| **skill_md 어떤 web 경로로도 미수신** | trust 경계·certifier decorrelation(6필드 화이트리스트) |
| **6필드 server-side catalog 검증** | enum 밖/미지 키 파이프라인 오염 차단 |

## 4. SES 정직성 게이트
Cognito-verified ≠ SES-verified(별개 목록). sandbox 에서 미검증 recipient 는 `MessageRejected` 로 조용히 버려짐 → **정직 표시 필수**: PUT /profile 가 UpdateItem 후 `_ensure_ses` → `delivery=active`("매일 {시}시 도착") | `verification_pending`("메일함의 SES 인증 클릭 후 발송"). **데모**: SES-verified 주소(moongons)로 가입·구독 → 즉시. sandbox 는 보안통제 아님(폭발반경 제한일 뿐).

## 5. 크로스레인 (LANE-A) + 검증
- **Cognito PublicClient `RequirePKCE: true`** — 미설정 시 code_verifier 없는 교환도 허용 → PKCE 서버 강제 안 됨. **LANE-A 요청(CFN 1줄)**, 공개 전 필수(데모는 저위험 허용). → 핸드셰이크 문서에 추가.
- 배포 후 검증: id_token→200 · access_token→**401**(aud 없음) · 토큰 없음→401 · `OPTIONS /profile`→200+CORS(preflight). timezone zoneinfo 검증. 신규 구독자 GET /profile→`{subscribed:false, recipient:email, profile:{}}`.
