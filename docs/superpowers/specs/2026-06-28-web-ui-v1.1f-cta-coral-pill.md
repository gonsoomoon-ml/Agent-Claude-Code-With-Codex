# ④ Web UI — v1.1f CTA 시그니처 코랄 pill 설계 스펙 (2026-06-28, LANE B)

> 증분: 레퍼런스(다크 프리미엄·터미널 미학 랜딩)의 시그니처 **코랄 pill CTA**를 우리 퍼널에 적용.
> 사용자 결정: **A(CTA-only)** — 라이트 레이아웃·정보구조 유지, **버튼 위치·스타일만** 레퍼런스화. + v1.1d에서 미룬 **스피너 회전(`@keyframes spin`)** 동봉.
> 범위: **프론트엔드 전용**(`web/`). 백엔드·라우팅·데이터흐름·검증 로직 무변경.
> 근거: ux-redesign-proposal 워크플로(6 에이전트, minimal-cta 추천 — TRUST 제품이라 코랄을 *화면당 행동 1개*에만, 증거 표면은 중립 유지).

## 0. 결정
- **`theme.ts` seam 신규** — `coralPill`/`ghostPill`/`coralDisabled` 토큰 한곳. forward-compatible(후속 B 다크히어로로 확장 가능한 진입점).
- **/setup 액션행 재배치**: 체험하기를 **1번째 primary 슬롯**으로(현재 2번째·회색·비활성처럼 보임 → 위계 부여). 옆 버튼(authed=`구독하기` / 비로그인=`로그인 / 구독하기`)은 **ghost(테두리만) pill**로 강등. → 화면당 코랄 액션 1개.
- **랜딩 CTA**: `구독 설정하기 →` 파란 사각 → **코랄 pill**(네비게이션이므로 ▶ 대신 **trailing →** 유지).
- **시그니처 코랄 pill 스타일**: rounded-full(radius 9999) · 코랄 그라데이션 `linear-gradient(135deg,#FF9E80 0%,#FF6B47 100%)` · 텍스트 **진한 웜 잉크 `#1A0F0A`**(흰색 아님 — 레퍼런스의 정의적 고대비 디테일) · **선행 `▶`**(`aria-hidden`) · 16px/600 · padding 14×28 · hover=밝기 +8%+코랄 글로우+살짝 부상 · disabled=회색(`#ececec`/`#aaa`, ▶ 숨김).
- **라이트 캔버스·정보구조·검증 증거(샘플 메일 iframe) 무변경** — "검증 후 발행" 신뢰 브랜드 보호.
- **불변식(비협상)**: `handleTrial`/`handleSubscribe`/`startLogin` 핸들러 + 체험 disabled 게이트(이메일 정규식 + 출처 ≥1) **verbatim 유지**. 접근성 이름 `/체험하기/`·`/구독하기/` 유지(▶/→는 `aria-hidden` → 기존 vitest regex 매처 green).

## 1. 컴포넌트

### 1a. `web/src/theme.ts` (신규, ~35줄)
- `colors`: `coralFrom`#FF9E80 · `coralTo`#FF6B47 · `coralInk`#1A0F0A · `ghostBorder`rgba(0,0,0,0.15) · `ghostText`#444.
- 스프레드 가능한 스타일 객체 3개:
  - `coralPill`: `display:inline-flex, alignItems:center, gap:8, borderRadius:9999, padding:'14px 28px', fontSize:16, fontWeight:600, color:coralInk, background:'linear-gradient(135deg,#FF9E80 0%,#FF6B47 100%)', border:'1px solid transparent', cursor:'pointer', textDecoration:'none', boxShadow:'0 1px 2px rgba(0,0,0,0.08)'`.
  - `ghostPill`: 동일 base + `padding:'12px 22px', fontSize:14, color:'#444', background:'transparent', border:'1px solid rgba(0,0,0,0.15)'`.
  - `coralDisabled`: `background:'#ececec', color:'#aaa', boxShadow:'none', cursor:'not-allowed'`.
- base look은 inline(여기), **pseudo-state(hover/disabled glyph)는 `index.html`**.

### 1b. `web/src/pages/Form.tsx`
- `import { coralPill, ghostPill, coralDisabled } from '../theme'`.
- `trialDisabled` 상수 호이스트(현 인라인 정규식 `isTrialSubmitting || !(selected.length && /…/.test(email))` 중복 제거 — DRY).
- 액션행 flex(현 L176-203)에 `alignItems:'center', flexWrap:'wrap'` + **순서 재배치**:
  - **체험하기 = first**: `className="cta-coral"`, `style={{...coralPill, ...(trialDisabled?coralDisabled:null)}}`, `disabled={trialDisabled}`, 내용 `{isTrialSubmitting ? '보내는 중…' : <><span aria-hidden="true">▶</span> 체험하기</>}`.
  - **구독/로그인 = second**: `className="cta-ghost"`, `style={ghostPill}`, label 끝에 `<span aria-hidden="true">→</span>`.
  - 핸들러·disabled 분기 verbatim.

### 1c. `web/src/pages/Landing.tsx`
- `구독 설정하기 →` `<Link to="/setup">`의 파란 인라인 style → `style={coralPill}` + `className="cta-coral"`. label의 trailing ` →` 유지. 구조 무변경.

### 1d. `web/index.html`
- (현재 빈) `<head>`에 `<style>` 추가:
  - `.cta-coral:hover:not([disabled]) { filter:brightness(1.08); box-shadow:0 0 24px rgba(255,107,71,0.35); transform:translateY(-1px); }` + `.cta-coral { transition:all .15s; }`
  - `.cta-coral[disabled] > [aria-hidden] { display:none; }`(disabled 시 ▶ 숨김)
  - `.cta-ghost:hover { border-color:rgba(0,0,0,0.45); color:#111; background:rgba(0,0,0,0.03); }`
  - **`@keyframes spin { to { transform:rotate(360deg); } }`** ← v1.1d ProgressModal `animation:'spin 1s …'` 가 이걸 못 찾아 ⏳ 안 돌던 문제 해결.

## 2. 테스트 (vitest, web)
- **기존 Form 테스트 green 유지**: ▶/→가 `aria-hidden` 이라 접근이름 `/체험하기/`·`/구독하기/` 매처 무손상 · 체험 disabled 게이트(이메일+출처) 동작 무변경.
- **신규**: 액션행에서 (a) 체험하기 버튼이 **첫 자식** + `class="cta-coral"` · (b) 사이드 버튼 `class="cta-ghost"` · (c) 이메일 무효/출처 0 → 체험 `disabled`, 이메일 유효+출처≥1 → enabled(게이트 보존).
