# ④ Web UI — v1.1g 미디어 선택 카드 그리드 설계 스펙 (2026-06-28, LANE B)

> 증분: 미디어 선택(`SourcePicker`)을 카테고리별 체크박스 리스트 → **반응형 카드 그리드**로. 동기: 사용자가 분야(area)·미디어를 계속 추가 예정 → 확장되는 N×M 카드 레이아웃.
> 사용자 결정: **Scope B(폴리시 카드 그리드)** — 미니멀 카드 그리드 + hover/focus·cap 강조·선택 그림자.
> 범위: **프론트엔드 전용**. `toggleSource`(선택 권위)·카운터·MAX-5·데이터흐름 무변경. 데이터 주도(카탈로그에 추가 시 자동 reflow).
> 근거: sourcepicker-card-grid 워크플로(5 에이전트) → category-sections-grid 추천(분야 멘탈모델 1:1·최저 위험·테스트 전부 보존).

## 0. 결정
- **카테고리 섹션 유지** + 각 섹션 안에 **반응형 카드 그리드**(`repeat(auto-fill, minmax(150px,1fr))`). 분야 추가=섹션 1개, 미디어 추가=카드 1개 → 자동 확장.
- **카드 = `<label className="src-card">`가 시각적으로 숨긴 native `<input type="checkbox">`를 감쌈** (sr-only **clip** 패턴, `display:none` 금지). `aria-label={s.name}` + native `disabled` + `toggleSource` → **기존 테스트 전부 보존**(`getByLabelText`·checkbox role·MAX-5·카운터).
- **데이터는 `{key,name,lang}`뿐**(로고/설명 없음) → 카드 시각 정체성 = **이름 첫 글자 아바타**(`name.trim().charAt(0)`). 로고/설명은 후속 카탈로그 확장 시 카드가 자라는 구조.
- **코랄은 선택 상태에만**(v1.1f `theme.ts` 토큰 재사용). 미선택은 완전 중립 → 코랄=명확히 "선택됨".
- **카운터 sticky** + cap 도달 시 코랄 강조. 문자열은 **byte-identical** 유지.

## 1. 컴포넌트

### 1a. `web/src/theme.ts` (토큰 추가)
- `colors`에 `coralWash: 'rgba(255,107,71,0.06)'` 추가(선택 카드 배경). 나머지 코랄(coralFrom/coralTo/coralInk)·아바타 그라데이션은 기존 토큰 재사용(`linear-gradient(135deg, ${colors.coralFrom}, ${colors.coralTo})`). coralPill(v1.1f) 무변경.

### 1b. `web/src/components/SourcePicker.tsx` (재작성)
- 상단 카운터 div: `position:sticky; top:0; background:#fff; zIndex:1`, 문자열 `선택 {selected.length} / {max} (최대 {max}개)` **그대로**. `atLimit`(=cap) 이면 color=coralTo·fontWeight 600.
- 카테고리별 `<fieldset><legend>{cat.name}</legend>` 유지(섹션). 안에 `<div className="src-grid">`(grid CSS는 인라인: `display:grid; gridTemplateColumns:'repeat(auto-fill, minmax(150px, 1fr))'; gap:10`).
- 각 소스 = `<label className="src-card" style={cardStyle(checked)}>`:
  - 숨긴 checkbox: `<input type="checkbox" aria-label={s.name} checked={checked} disabled={!checked && atLimit} onChange={() => onChange(toggleSource(selected, s.key, max))} style={SR_ONLY} />` (SR_ONLY=clip 패턴).
  - 아바타 32px 원(`name.trim().charAt(0)`): 미선택 `#f3f3f3`/`#666`, 선택 시 coral 그라데이션 + coralInk 글자. `aria-hidden`.
  - 이름 14px/600 한 줄(`overflow:hidden; textOverflow:ellipsis; whiteSpace:nowrap`).
  - lang 배지 `({s.lang})` 대문자 10px #999(선택돼도 중립).
  - ✓ 우상단 절대배치 18px 코랄 그라데이션 배지 — `checked`일 때만 렌더, `aria-hidden`.
  - `cardStyle(checked)`: 미선택=흰 배경/`1px solid #eee`/radius12/padding'12px14px'; 선택=`1.5px solid coralTo`+`background coralWash`+`boxShadow 0 1px 3px rgba(255,107,71,0.18)`. `atLimit && !checked` → opacity .45 + cursor not-allowed.

### 1c. `web/index.html` (전역 `<style>` — 인라인 불가한 pseudo만)
- `.src-card { transition: box-shadow .12s; }`
- `.src-card:hover { box-shadow: 0 1px 4px rgba(0,0,0,0.12); }` — 미선택 카드 hover lift. (⚠️ 선택 카드는 **인라인 boxShadow**(코랄)가 specificity로 이 규칙을 이김 → 선택 카드는 코랄 그림자 유지. 카드 border/배경도 전부 인라인이라 hover 를 border가 아닌 **box-shadow**로 거는 이유.)
- `.src-card:focus-within { outline: 2px solid #FF6B47; outline-offset: 2px; }` — 키보드 포커스(outline은 인라인에 없어 적용됨).
