> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** 체험하기 클릭 후 "정지 화면" 제거 — 버튼 loading + 스피너/경과타이머 상태 카드 + runtime 이 단계별 status 를 기록(`GET /trial/status` 폴링)해 카드가 "생성 중 → ✅ 발송됨"까지 실시간 갱신.

**Architecture:** runtime mode=trial 이 `status_fn`(주입)으로 briefing-trials 에 status 기록(generating/sent/fallback/expired/failed) → `GET /trial/status?email=`(public) → 프론트 3초 폴링 + 순수 `trialStatusMessage` 매핑 + 단말 정지.

**Tech Stack:** Python(boto3 dynamodb) · FastAPI · React/TS · pytest · vitest.

## Global Constraints

- `status` 는 **DDB 예약어** → ExpressionAttributeNames `#s` 우회.
- runtime status 쓰기 = **DI `status_fn`**(테스트는 fake); 부수효과는 entrypoint 가 배선. dry_run → status_fn no-op.

- 폴링: 3초, **단말 상태에서 정지**, 전체 ~10분 타임아웃.
- 기존 v1.1a/v1.2 동작·테스트 무손상.

```python
def run_trial(settings, store, card_cache, payload, *, ses, run_briefing_fn, deliver_fn,
              fallback_fn, sleep_fn, run_date, attempts=45, sleep_seconds=20,
              status_fn=lambda s, p=None: None):
    email = payload["email"]
    if not poll_verified(ses, email, attempts=attempts, sleep_seconds=sleep_seconds, sleep_fn=sleep_fn):
        status_fn("expired")
        return f"trial timeout(unverified): {email}"
    status_fn("generating")
    user = build_trial_user(payload, settings)
    briefs = run_briefing_fn(settings, store, [user], card_cache=card_cache, ledger=None, run_date=run_date)
    if not briefs:
        fallback_fn(email); status_fn("fallback")
        return f"trial empty: {email}"
    b = briefs[0]
    if b.published > 0:
        deliver_fn(b); status_fn("sent", b.published)
        return f"trial delivered({b.published}): {email}"
    fallback_fn(email); status_fn("fallback")
    return f"trial fallback(published=0): {email}"
```

```python
        import boto3  # (이미 branch 에 있음)
        trials = boto3.resource("dynamodb", region_name=settings.region).Table("briefing-trials")
        def _status(s, p=None):
            if dry:
                return
            names = {"#s": "status"}; vals = {":s": s}; expr = "SET #s = :s"
            if p is not None:
                expr += ", published = :p"; vals[":p"] = p
            try:
                trials.update_item(Key={"email": email}, UpdateExpression=expr,

            except Exception as e:  # noqa: BLE001

                app.complete_async_task(task_id)
```

```typescript
// web/src/lib/trialStatus.ts
const TERMINAL = new Set(['sent', 'fallback', 'expired', 'failed'])
export function isTerminal(s: string): boolean { return TERMINAL.has(s) }
export function trialStatusMessage(status: string, published?: number): { text: string; done: boolean } {
  switch (status) {
    case 'verification_pending': return { text: '📧 확인 메일의 링크를 클릭하세요 (받은편지함/정크 확인).', done: false }
    case 'sending':
    case 'generating': return { text: '⏳ 브리핑 생성 중… (보통 1~5분). 이 창은 닫으셔도 됩니다.', done: false }
    case 'sent': return { text: `✅ 발송 완료! ${published ?? ''}건 — 받은편지함을 확인하세요.`, done: true }
    case 'fallback': return { text: '오늘은 검증 통과한 새 소식이 부족해 안내 메일을 보냈어요.', done: true }
    case 'expired': return { text: '확인 시간이 지났어요 — 다시 시도해주세요.', done: true }
    case 'failed': return { text: '문제가 생겼어요 — 잠시 후 다시 시도해주세요.', done: true }
    default: return { text: '처리 중…', done: false }
  }
}
```
