# CLAUDE.md

Guidance for Claude Code (and Codex) when working in this repository.

## Language / 언어

- Write **all documentation, docstrings, and code comments in a Korean-friendly way (한국어 우선)** so Korean readers can follow easily.
  - **Applies to:** Markdown docs (`*.md`), design specs, READMEs, module/function/class docstrings, and inline code comments.
  - **Keep code itself in English:** identifiers, function/variable/class names, API names, CLI flags, file paths, and log messages stay in English for portability.
  - When a precise English technical term helps, write it **bilingually** — e.g. `검증 후 실행 게이트(verify-before-fire gate)`, `의무 원장(obligation ledger)` — so the meaning is clear in both languages.
